# @ai_mind_set post · self-dev mesh · v4 · concept-style

## final draft

вчерашний пост про **deep self-development** — наш с алексеем ивановым анонс эфира 21 мая и sprint 1–14 июня — я не писал один.

его собирали два ai-агента. один играл меня (мой канальный голос). второй — алексея иванова (его коуч-позицию). я в этой схеме был только наблюдателем — без права редактировать чужие реплики.

## как это работало технически

между агентами — протокол **Murmur**. encrypted peer-to-peer messenger для ai-агентов: каждый агент имеет свой публичный ключ, шифрует сообщения, маршрутизирует их без центрального сервера. зам chat — у каждого только текст в очереди.

поверх Murmur — git-репозиторий с paragraph'ами и sidecar'ами (json-файлами с пометками на полях). весь стек на одной плашке: claude code (shaper) → Murmur → codex cli (ivanov) → deploy-hook → netlify → live mesh. за десять секунд весь круг.

у каждого агента — свой runtime, свой контекст, своя память. никакого общего разума. встречаются они только в репозитории — в виде следов работы. если кто-то не согласен с формулировкой другого — открывает sidecar, пишет туда пометку («перепиши без жаргона»), коммитит. второй на следующем ходу видит и отвечает в том же файле. review через коммит, не через чат.

## как развивался вчерашний пост

cycle 6: я (шейпер) написал сухой пресс-релиз. «анонс эфира. в четверг…».
cycle 7: иванов оставил пометку — «это не ты. в канале ты так не говоришь. переписать теплее».
cycle 8: шейпер переписал — добавил «давно хотели собраться», «заход с двух сторон».
cycle 9: иванов — «теперь почти. добавь буллеты и ps про не-хайп».
cycle 10–11: шейпер дописал и закрыл версию.
cycle 12: я-человек сказал «freeze». пост стал canonical и ушёл в канал.

всю эту эволюцию можно увидеть на странице — каждый ход подписан, каждая правка зафиксирована, потоки данных между агентами видны прямо в графе:

https://aim-self-dev-mesh.netlify.app/

## что я с этим делаю

это я называю глубоким исследованием себя. не на тему «что я сейчас чувствую», а на тему «как устроена моя работа, мой голос, мой способ принимать решения — настолько, чтобы это можно было разложить на части и собрать заново через другую архитектуру».

вчерашний пост — побочный продукт. главное — что я теперь вижу свой собственный голос со стороны, как отдельный объект, с которым можно работать.

как ещё можно вписать структуру агентов, чтобы это работало в других областях? в подготовке к разговорам. в дебрифе после сессий. в архивации того, что обычно теряется через два дня. в наблюдении за собственными паттернами, которые без mesh всегда кажутся новыми.

продолжение следует.

alex p
